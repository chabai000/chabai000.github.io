3. 启动JVM

4. 类加载（Class Loading）

JVM开始执行MyProgram时，类加载器负责加载所需的类。

引导类加载器（Bootstrap Class Loader）

加载核心Java类库（例如，java.lang.*）。

扩展类加载器（Extension Class Loader）

加载Java扩展库（例如，javax.*）。

应用程序类加载器（Application Class Loader）

加载应用程序类路径上的类（例如，MyProgram.class）。

5. 字节码验证（Bytecode Verification）

在类加载的过程中，JVM会对加载的字节码进行验证。

文件格式验证

检查文件格式的正确性。

字节码验证

确保字节码指令的合法性。

语义验证

验证字节码的操作是否合法，变量是否初始化等。

数据流验证

确保程序执行不会出现栈溢出或下溢等问题。

6. 准备（Preparation）

JVM为类的静态变量分配内存，并将其初始化为默认值。

7. 解析（Resolution）

JVM将符号引用（符号化的变量和方法名）替换为直接引用（内存地址）。

8. 初始化（Initialization）

执行类的初始化代码，即执行静态初始化块和静态变量的初始化赋值。

9. 执行主方法（Main Method Execution）

JVM寻找程序的入口点，即public static void main(String[] args)方法，并开始执行该方法。

10. 执行引擎（Execution Engine）

JVM的执行引擎负责执行字节码指令。

解释器（Interpreter）

逐行解释执行字节码指令。

即时编译器（JIT Compiler）

将热点代码（频繁执行的代码）编译成机器码，提高执行效率。

11. 内存管理

JVM使用堆和栈管理内存。

堆（Heap）

所有对象实例和数组都存放在堆中，由所有线程共享。

栈（Stack）

每个线程都有自己的栈，存放方法调用的栈帧。每个栈帧包含局部变量表、操作数栈、动态链接和返回地址等。

堆和栈扩展

在Java中，堆和栈的分配方式不同，使用的目的也不同。堆用于存储动态分配的对象实例和数组，而栈用于存储方法的调用信息。

12. 垃圾回收（Garbage Collection）

JVM定期执行垃圾回收，回收不再被引用的对象，释放内存空间。

13. 程序结束

当main方法执行完毕，JVM会进行善后工作，包括调用finalize方法（如果有），进行最后一次垃圾回收，释放占用的资源，最终终止程序运行。

总结

  1. 编写并编译Java源代码：生成字节码文件。
  2. 启动JVM并加载类：通过类加载器加载应用程序的类。
  3. 字节码验证：确保字节码的安全性和正确性。
  4. 准备：为静态变量分配内存并初始化为默认值。
  5. 解析：将符号引用替换为直接引用。
  6. 初始化：执行类的静态初始化块和静态变量初始化赋值。
  7. 执行主方法：JVM寻找并执行main方法。
  8. 执行引擎：解释或编译字节码。
  9. 内存管理：使用堆和栈管理内存。
  10. 垃圾回收：回收不再被引用的对象。
  11. 程序结束：进行善后工作，调用finalize方法，释放资源。
